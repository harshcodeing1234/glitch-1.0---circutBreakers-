# 🚀 Project Pulse - Task Management System

A modern, responsive web application designed to solve task distribution confusion in group projects through transparent task claiming and real-time collaboration.

---

## 🌟 Problem Statement

### The Challenge
In group projects, team members often face:
- **Task Distribution Confusion** - "Who is doing what?"
- **Duplicate Work** - Multiple people working on same task
- **Missed Deadlines** - Tasks left unassigned or forgotten
- **Poor Communication** - No central place to track progress
- **Accountability Issues** - Unclear ownership of responsibilities

### Real-World Impact
- 📉 **Reduced Productivity** - Time wasted on coordination
- 😤 **Team Frustration** - Confusion leads to conflicts
- ⏰ **Project Delays** - Poor task management affects timelines
- 📊 **Quality Issues** - Rushed work due to last-minute discoveries

---

## 💡 Solution

**Project Pulse** provides a centralized platform where:
- ✅ **Clear Task Ownership** - Visual indication of who claimed what
- 🔄 **Real-time Updates** - Instant synchronization across team
- 📊 **Progress Tracking** - Dashboard with project statistics
- 👥 **Team Collaboration** - Transparent task distribution
- 📱 **Easy Access** - Web-based, works on any device

### How It Solves The Problem
1. **Eliminates Confusion** - Clear task status (claimed/unclaimed)
2. **Prevents Duplication** - One person per task system
3. **Improves Accountability** - Name attached to each claimed task
4. **Enhances Communication** - Central hub for all project tasks
5. **Increases Efficiency** - Quick task claiming and tracking

## 🚀 Quick Start

### Prerequisites
```bash
Node.js (v14+) and npm
```

### Installation
```bash
# Navigate to project
cd "glitch 1.0/landing page"

# Install dependencies
npm install

# Start application
npm start
```

### Usage
1. Open `http://localhost:3001`
2. Register/Login to your account
3. Start creating and claiming tasks
4. Monitor progress on dashboard

## 🏗️ Architecture

### **System Design**
```
Frontend (HTML/CSS/JS) ↔ Express Server ↔ SQLite Database
```

### **Database Schema**
- **Users** - Authentication and profiles
- **Tasks** - Task information and assignments
- **Settings** - User preferences

### **API Endpoints**
- `/api/auth/*` - Authentication routes
- `/api/tasks/*` - Task management
- `/api/dashboard/*` - Statistics and analytics
- `/api/users/*` - User management
